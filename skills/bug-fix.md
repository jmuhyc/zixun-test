# Bug Fix SOP - Bug 修复标准流程

> 用于系统性修复 Bug，确保问题被彻底解决且不引入新问题。

## 前置条件

- [ ] Bug 现象已记录
- [ ] 复现步骤已明确
- [ ] 相关代码已定位

## 执行步骤

### 1. 理解问题

1. 记录 Bug 现象
2. 确定复现步骤
3. 定位问题代码
4. 分析根本原因

### 2. 修复问题

1. 制定修复方案
2. 实施修复
3. 添加防御性检查（如果适用）

### 3. 验证修复

**复现测试**:
```bash
# 执行复现步骤
python <affected_script.py>

# 确认 Bug 已修复
```

**回归测试**:
```bash
# 运行相关测试
python -m pytest tests/

# 确认无副作用
```

### 4. 更新文档

- [ ] 如果 Bug 来自配置问题，更新 [docs/api.md](docs/api.md)
- [ ] 如果是新发现的问题，记录到 [MEMORY.md](MEMORY.md)
- [ ] 如果是规则遗漏，更新 [AGENTS.md](AGENTS.md)

## 检查清单

- [ ] Bug 现象已消除
- [ ] 复现步骤通过
- [ ] 相关功能未受影响
- [ ] 代码格式符合规范
- [ ] 无引入新问题

## 规则引用

- 遵循 [AGENTS.md](AGENTS.md) 中的禁止事项
- 不要修改 `config.py`
- 修复后运行代码审查
