# Code Review SOP - 代码审查标准流程

> 用于对代码变更进行系统性审查，确保质量和一致性。

## 前置条件

- [ ] 代码变更已完成
- [ ] 本地测试通过
- [ ] 代码格式符合 Black 规范

## 执行步骤

### 1. 准备审查

```
# 确保代码已格式化
black <changed_files>

# 确保导入已排序
isort <changed_files>
```

### 2. 审查清单

#### 2.1 功能正确性
- [ ] 代码逻辑是否正确实现需求
- [ ] 边界条件是否处理
- [ ] 错误处理是否完善

#### 2.2 代码质量
- [ ] 遵循 PEP 8 规范
- [ ] 变量命名清晰有意义
- [ ] 函数职责单一
- [ ] 无重复代码

#### 2.3 安全检查
- [ ] 无硬编码 API 密钥
- [ ] 无敏感信息泄露
- [ ] 输入验证完善

#### 2.4 性能考虑
- [ ] 无明显的性能问题
- [ ] 资源使用合理

### 3. 审查输出

```
## Code Review 报告

### 变更文件
- file1.py
- file2.py

### 发现问题
- [问题描述] @负责人

### 建议
- [改进建议]

### 结论
[通过/需要修改]
```

## 验证方法

1. 运行 `black --check <files>` 确认格式
2. 运行 `isort --check <files>` 确认导入
3. 本地执行相关功能测试

## 规则引用

- 遵循 [AGENTS.md](AGENTS.md) 中的代码风格规范
- 禁止修改 `config.py`
